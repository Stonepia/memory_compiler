## Important implementation notes

- Imports `FailureMode`, `PrecheckRequest`, `PrecheckResult`, `PrecheckRule`
  from `src/dmc/schemas.py` and `DMCStore`/`DMCError` from `src/dmc/store.py`;
  no shapes redefined in `precheck.py`.
- Built-in rules (deterministic):
  1. `failure-mode-resemblance` (warn) — `match_failure_modes` scans stored
     failure-mode objects (`objects/failure_modes/`, written via
     `store.write_object("failure_modes", id, FailureMode)`) and matches on
     trigger-phrase containment or >=2 distinctive shared tokens (length >= 4,
     stopword-filtered). Sorted by id; malformed files skipped, never raised.
  2. `benchmark-claim-without-artifact` (warn) — fires when action/intent/
     command mention perf/benchmark/speedup/etc. and `task_context` has no
     benchmark-artifact reference (no `benchmark_artifact`/`artifact(s)`/
     `evidence` key and no `dmc://artifact` URI).
  3. `edit-without-task-ref` (warn) — fires for edit-like actions with no
     task/plan reference in `task_context`.
  4. `direct-skill-mutation` (**block**) — fires when an edit targets an
     accepted skill path (`.dmc/skills/**`, `skills/tier*`) and is not routed
     through the proposal path; surfaces the proposal-path hint as required
     evidence.
  5. `memory-write-without-evidence` (warn) — extra deterministic rule so
     missing evidence for durable memory/knowledge writes is never *silently*
     allowed (honours the card's forbidden shortcut). Aggregates required
     evidence.
- Decision precedence: any block -> `block`; else any warn -> `warn`; else
  `allow`. `matched_rules` holds fired rule ids, `warnings`/`reasons` hold
  human-readable messages, `required_evidence_before_commit` aggregates evidence
  from fired rules (deduped, order-preserving).
- `load_precheck_rules` always returns the built-ins and optionally merges extra
  rules stored under `objects/precheck_rules/` (sorted by id, built-in ids win).
  No LLM is consulted.
