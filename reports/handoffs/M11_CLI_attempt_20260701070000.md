## Important implementation notes

- Business logic stays in core (module-specific forbidden shortcut respected).
  Command bodies only marshal files/args and call `planner`, `renderer`,
  `retriever`, `precheck`, `recorder`, `distiller`, and `DMCStore`.
- DMC root resolution mirrors the MCP server: explicit `--dmc-root` else cwd.
  `.dmc` is created via `DMCStore.initialize()`.
- Failures raise a non-zero `typer.Exit` with a clear stderr message via
  `_fail`. `--out` writers create parent directories.
- `precheck` uses exit code `2` for a `block` decision (distinct from `1` for
  validation/IO errors) so callers can gate on it; `warn`/`allow` exit `0`.
