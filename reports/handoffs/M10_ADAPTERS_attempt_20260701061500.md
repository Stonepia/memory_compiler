## Summary

Implemented `src/dmc/adapters.py` — adapter bundle generation for Codex,
Copilot, and OpenCode. Each `render_<target>_bundle(project_root)` returns a
`dict[str, str]` of relative path -> static, protocol-only file content (no
dynamic project facts embedded, per module contract). `export_agent_bundle`
writes a bundle to disk and returns the written `list[Path]`, requiring an
explicit `out_dir` (never defaulting to the resolved project root) and
resolving the project root from an explicit `project_root`, a `store.root`
attribute, or `cwd()` as a last resort. Also wired the previously-placeholder
`dmc export-agent-bundle` CLI command and fixed a `Path`-serialization gap in
M09's lazy MCP wrapper now that M10 is real.

### Revision (PR #3 review, CHANGES_REQUIRED)

1. **Codex root override was fake.** The generated `.codex/config.toml.template`
   referenced `env = { DMC_ROOT = "." }`, but `dmc serve`/`resolve_root` never
   read `DMC_ROOT` — only an explicit `--root` argument. Fixed by using the
   real, already-implemented `dmc serve --root .` flag in the template.
2. **`export_agent_bundle` silently defaulted to project root.** The original
   `out=None` default meant a codex/opencode export with no explicit output
   directory would write straight into the resolved root — which would
   overwrite this repo's own root `AGENTS.md` if ever run there. Fixed by
   renaming `out` -> `out_dir` (now required; matches the module card's
   contract), and adding `default_out_dir(project_root, target)` — a
   `.dmc/adapters/generated/<target>` staging path — used by the CLI
   (`--out` optional) and the MCP wrapper (`out` optional) whenever no
   explicit output directory is supplied.

## Important implementation notes

- Protocol content is shared via a single `_PROTOCOL_STEPS` string reused
  across all three targets' `AGENTS.md`/instructions/skill files, so the
  seven required obligations (plan/brief at start, use external MCPs,
  precheck before risk, record after action, commit at checkpoint, distill at
  end, never mutate skills directly) stay in lock-step across surfaces.
- `project_root` is accepted by every `render_*` function for API symmetry
  and future extension, but its value is never interpolated into generated
  text — this preserves "no dynamic facts in stable instruction files"
  (`AGENTS.md` / Copilot instructions / OpenCode `AGENTS.md`).
- `export_agent_bundle`'s `store` parameter only reads `.root`; it does not
  call any `DMCStore` methods, keeping the generator a pure file writer with
  no store-schema coupling.
- Fixed a real (if latent) bug in M09's `dmc_export_agent_bundle`: it
  previously passed `export_agent_bundle`'s `list[Path]` result straight into
  the envelope, which is not JSON-serializable through FastMCP. It is now
  converted to `list[str]`.
- `export_agent_bundle` never guesses an output directory: `out_dir` is a
  required parameter, and passing `None` explicitly raises `ValueError`
  rather than falling back to the project root.
