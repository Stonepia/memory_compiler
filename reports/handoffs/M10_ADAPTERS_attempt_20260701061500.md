## Important implementation notes

- Protocol content is shared via a single `_PROTOCOL_STEPS` string reused
  across all three targets' `AGENTS.md`/instructions/skill files, so the
  seven required obligations (plan/brief at start, use external MCPs,
  precheck before risk, record after action, commit at checkpoint, distill at
  end, never mutate skills directly) stay in lock-step across surfaces.
- `project_root` is accepted by every `render_*` function for API symmetry
  and future extension, but its value is never interpolated into generated
  text — this preserves "no dynamic facts in stable instruction files"
  (`AGENTS.md` / Copilot instructions / OpenCode `AGENTS.md`).
- `export_agent_bundle`'s `store` parameter only reads `.root`; it does not
  call any `DMCStore` methods, keeping the generator a pure file writer with
  no store-schema coupling.
- Fixed a real (if latent) bug in M09's `dmc_export_agent_bundle`: it
  previously passed `export_agent_bundle`'s `list[Path]` result straight into
  the envelope, which is not JSON-serializable through FastMCP. It is now
  converted to `list[str]`.
