## Important implementation notes

- `Slug` = `Annotated[str, AfterValidator(...)]` enforcing non-empty slug-like
  ids (`^[A-Za-z0-9][A-Za-z0-9._-]*$`). Reused for every id field.
- `Uri` enforces `<scheme>://<path>` form with scheme in `KNOWN_URI_SCHEMES`
  (dmc, plan, session, event, artifact, episode, failure_mode, proposal,
  eval_case, knowledge, skill, file, git, http, https).
- `Provenance` accepts a bare URI string (coerced to `{"source": uri}`) so the
  sample `provenance: [session://sess_demo]` shape validates.
- Durable/evidence-bearing memory objects (`TraceEvent`, `KnowledgeRef`,
  `ArtifactCard`, `EpisodeCard`, `FailureMode`, `SkillUpdateProposal`,
  `EvalCase`) use `NonEmptyProvenance` (`list[Provenance]`, `min_length=1`);
  empty or missing provenance is rejected.
- `TraceEvent` aligns with `templates/trace_event.schema.json`; requires
  `session_id`, `event_id`, `phase`, `action.kind`, `observation.outcome`,
  `timestamp`, and non-empty `provenance`.
- `PlanGraph` aligns with `templates/plan_graph.schema.json` and validates at
  the graph level: unique node ids, every dependency references an existing
  node, no self-dependency.
- `EvalCase` aligns with `templates/eval_case.schema.json`; requires task,
  source_session, outcome, labels, provenance.
- `AgentState` is intentionally permissive (extra allowed) as it mirrors the
  coordination file owned by the state protocol, not a single module.
- No duplicate data shapes are defined elsewhere; this is the only contract.
