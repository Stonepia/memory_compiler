## Important implementation notes

- Imports all data shapes from `src/dmc/schemas.py` (`TraceEvent`,
  `ProjectState`, `ArtifactCard`, `SearchResult`); no shapes redefined.
- URI scheme is `dmc://<kind>/<id>` (the `dmc` scheme is in
  `KNOWN_URI_SCHEMES`, so returned URIs validate against schema `Uri`).
  `read_object` resolves URIs produced by `write_object`,
  `save_artifact_card`, `upsert_project_state` (`dmc://project_state/current`)
  and `append_event` (`dmc://event/<id>`, scanned from `events.jsonl`).
- Append-only: `append_event` and `save_artifact_card` open their JSONL files
  in `"a"` mode and never rewrite prior lines (asserted in tests).
- SQLite/FTS5: `search_index` virtual table with `uri/kind/scope` UNINDEXED
  (filter-only) and `title/body` indexed. Scope filtering via `WHERE scope IN
  (...)`; ranking via `bm25()` (reported `score = -bm25`, higher = better).
  Query terms are tokenized and quoted to avoid FTS syntax injection.
- `rebuild_index()` wipes and rebuilds the FTS index purely from files; tests
  delete `index.sqlite3` then rebuild and search successfully.
- `upsert_project_state` writes `state/project_state.yaml` (source of truth)
  and returns a monotonically increasing version int tracked in
  `project_state_meta`.
- Errors are explicit: invalid/unknown URIs raise `DMCValidationError` /
  `DMCNotFoundError` — never silent `None`.
