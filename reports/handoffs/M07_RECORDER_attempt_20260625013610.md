## Important implementation notes

- Shapes imported from `src/dmc/schemas.py` (`TraceEvent`, `ArtifactCard`,
  `TracePhase`); nothing redefined. `DMCStore` / `DMCValidationError` imported
  from `src/dmc/store.py`.
- `schemas.TraceEvent.phase` is already constrained to the required phase set
  via the `TracePhase` literal, so an event with a disallowed phase fails at
  construction (Pydantic `ValidationError`). `ALLOWED_PHASES` is derived from
  `get_args(TracePhase)` so recorder and schema cannot diverge.
- `schemas.TraceAction.kind` is an open non-empty string, so `record_event` is
  the authoritative gate for the allowed action-kind contract: an out-of-set
  `action.kind` raises `DMCValidationError` (never silently accepted). No
  schema tightening was needed (preferred per card/prompt).
- `record_event` returns the `dmc://event/<id>` URI from
  `store.append_event` (append-only `events.jsonl`).
- `record_artifact` persists via `store.save_artifact_card` (append-only
  `artifacts/index.jsonl`). With a `raw_path`, the raw file is copied to
  `.dmc/artifacts/raw/<id>/<filename>` and referenced on the card by
  `raw_artifact_path` (store-relative) and `raw_artifact_uri` (`file://`) only
  — raw bytes are never inlined into JSONL (asserted in tests).
- `session_events` delegates to `store.list_events(session_id)` preserving
  recorded (file) order.
- `summarize_session_trace` is deterministic (sorted count keys, sorted+deduped
  artifact refs). Dict shape is documented in the function docstring.
