## Public APIs implemented

```python
def record_event(event: TraceEvent, store: DMCStore) -> str: ...
def record_artifact(card: ArtifactCard, store: DMCStore, raw_path: Path | None = None) -> str: ...
def session_events(session_id: str, store: DMCStore) -> list[TraceEvent]: ...
def summarize_session_trace(session_id: str, store: DMCStore) -> dict: ...
```

`summarize_session_trace` returns a deterministic dict:
`{session_id, num_events, counts_by_phase, counts_by_action_kind,
counts_by_outcome, first_timestamp, last_timestamp, artifact_refs}`.

## Known limitations

- `artifact_refs` in `summarize_session_trace` are harvested from each event's
  `artifacts` mapping by collecting URI-like (`://`) string values; events that
  store artifact references under a non-URI shape are not counted.
- The recorder does not deduplicate `event_id`s; uniqueness/ordering is the
  store's append-only responsibility.

## Dependency changes
