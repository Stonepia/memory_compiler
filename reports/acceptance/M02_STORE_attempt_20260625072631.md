## Known limitations

- `search_text` ranking uses FTS5 `bm25()` defaults (no per-field weighting).
- Markdown objects are stored/parsed via YAML front matter; arbitrary prose
  bodies round-trip as `{"text": ...}` rather than structured dicts.
- `read_object` resolves the first matching extension in a fixed precedence
  order if multiple files share an object id (yaml > yml > json > md).
