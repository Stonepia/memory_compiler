## Scope completed

Implemented the complete local Typer CLI — the user-facing shell for V0 and the
stable interface for the integration tests. Every previously-placeholder command
(`state`, `plan`, `graph`, `brief`, `search`, `precheck`, `record`, `distill`)
is now wired to its owning DMC core function. No business logic lives in the
command bodies; each command loads/validates input, delegates to core, and
writes output.

Required commands from the module card, all present and functional:

```text
dmc state show
dmc state commit <patch-file>
dmc plan <task-file> --out <path>
dmc graph <plan-file> --format mermaid|markdown --out <path>
dmc brief <task-file> --out <path>
dmc search <query> --scope <scope>...
dmc precheck <action-file> --out <path optional>
dmc record <event-file>
dmc distill --session <session_id>
dmc export-agent-bundle --target codex|copilot|opencode --out <dir>
```

Required behavior:

- All commands return non-zero on validation/IO failure with a clear stderr
  error message (`_fail` helper).
- All file outputs create parent directories (`_emit_structured`,
  `_emit_text`, and the core `save_plan_graph` / `write_rendered`).
- DMC root defaults to the current working directory and is overridable with
  `--dmc-root`; `.dmc` is created inside it.

## Known limitations

- `dmc precheck` exits with code `2` (distinct from the code `1` used for
  validation/IO failures) when the decision is `block`, so a wrapper can gate
  on it. `warn`/`allow` decisions exit `0`. This is an additive, documented
  behavior beyond the card's minimum ("non-zero on validation failure").
- `dmc search` and `dmc brief` search only the local DMC store; there is no
  external/network retrieval in V0 (by design).
- Structured results are emitted as YAML; there is no `--format json` toggle
  (not required by the card).
