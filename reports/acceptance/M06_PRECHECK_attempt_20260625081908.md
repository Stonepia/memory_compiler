  path is not blocked.

  rule is `block`.

    warn rule was still evaluated, not short-circuited) and `decision == "block"`.
  - `test_warn_takes_precedence_over_allow` — exactly one warn rule fires and no
    block rule fires; asserts `decision == "warn"` (never `allow`).

## Known limitations

- Failure-mode resemblance uses deterministic token-overlap/phrase matching
  (no semantic similarity, by design — no LLM/embeddings in v0). Very short or
  highly generic triggers may under-match; this is intentional to avoid false
  positives.
- `load_precheck_rules` merges optional stored rules from
  `objects/precheck_rules/` but the engine evaluates only the built-in rule
  predicates; custom stored rules are surfaced but not auto-executed in v0.
